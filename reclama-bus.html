ReclamaBus – Sistema de Feedback do Transporte Público

← Voltar ao portfólio

Descrição do Projeto

O ReclamaBus é um sistema criado para dar voz aos usuários do transporte público de Niterói, permitindo o registro de reclamações, relatos de ocorrências e opiniões gerais sobre o serviço. O sistema coleta dados como linha, horário e detalhes do relato, mantendo as informações pessoais dos usuários protegidas e invisíveis ao público. Além de centralizar essas informações, o objetivo maior é auxiliar as autoridades responsáveis a identificar e corrigir falhas recorrentes, promovendo melhorias reais no transporte público da cidade.

Principais objetivos

  • Oferecer aos usuários um canal direto para registrar reclamações ou opiniões sobre o transporte.
  • Permitir que o público e os gestores visualizem os problemas mais citados.
  • Contribuir para decisões mais eficazes e investimentos direcionados na melhoria do transporte público.

Funcionalidades implementadas

  • Cadastro com validação de dados (nome, idade, telefone, senha)
  • Login seguro com verificação de e-mail e senha
  • Atualização e exclusão de conta pelo próprio usuário
  • Registro de ocorrência informando linha, categoria, horário e detalhes
  • Visualização de todas as ocorrências cadastradas, permitindo consultas pelos responsáveis
  • Proteção dos dados pessoais: somente as ocorrências ficam públicas, garantindo privacidade

Integração com banco de dados

  • Tabela de usuários: armazena credenciais e informações como idade, PCD, idoso ou funcionário
  • Tabela de ocorrências: armazena relatos detalhados dos usuários
  • Uso de Prepared Statements para prevenir injeção de SQL e garantir segurança nas consultas
  • Métodos CRUD (Create, Read, Update, Delete) para gerenciar dados dos usuários e ocorrências

Melhorias previstas

  • Implementar filtros para buscar ocorrências por linha ou categoria
  • Criar painel administrativo para gestores do transporte visualizarem estatísticas
  • Melhorar a interface para tornar o sistema mais intuitivo ao usuário

Tecnologias Utilizadas

  • Linguagem de programação: Java
  • Banco de dados: PostgreSQL
  • Arquitetura: Programação orientada a objetos, com classes e métodos específicos para usuários e ocorrências

Classes Principais

  • Usuarios: gerencia cadastro, login, atualização e exclusão de contas
  • Ocorrencias: gerencia o registro e exibição das reclamações
  • DbContext: gerencia a conexão e operações no banco de dados
